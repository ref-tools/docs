## ref_search_documentation

Search through Ref's indexed documentation using natural language queries.

### Parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `query` | string | Yes | Search query in natural language |
| `maxResults` | integer | No | Maximum results to return (default: 10, max: 50) |
| `sources` | array | No | Specific documentation sources to search |

### Usage Examples

### Custom Client Integration

For building custom integrations:

```javascript
class RefToolsClient {
  constructor(mcpClient) {
    this.mcp = mcpClient;
  }
  
  async searchDocs(query, options = {}) {
    return await this.mcp.callTool({
      name: 'ref_search_documentation',
      arguments: {
        query,
        maxResults: options.maxResults || 10,
        sources: options.sources
      }
    });
  }
  
  async readUrl(url, format = 'markdown') {
    return await this.mcp.callTool({
      name: 'ref_read_url',
      arguments: { url, format }
    });
  }
  
  async comprehensiveSearch(topic) {
    // Search documentation first
    const docs = await this.searchDocs(topic);
    
    // If we find specific URLs in results, read them for details
    const urlResults = [];
    for (const result of docs.results.slice(0, 3)) {
      const content = await this.readUrl(result.url);
      urlResults.push(content);
    }
    
    return { documentation: docs, details: urlResults };
  }
}
```

## Performance Considerations

### Credit Usage

| Tool | Credit Cost | Notes |
|------|-------------|-------|
| `ref_search_documentation` | 1 credit | Fast, indexed search |
| `ref_read_url` | 1 credit | External URL fetching |

### Response Times

- **Documentation Search**: ~200-500ms (indexed content)
- **URL Reading**: ~1-5 seconds (depends on target site)

### Best Practices

<CardGroup cols={2}>
  <Card title="Efficient Usage">
    - Use documentation search before URL reading
    - Cache results when possible  
    - Combine related queries
    - Use appropriate result limits
  </Card>
  
  <Card title="Error Handling">
    - Handle timeout scenarios gracefully
    - Provide fallback options
    - Validate URLs before reading
    - Monitor credit usage
  </Card>
</CardGroup>

## Troubleshooting

<AccordionGroup>
  <Accordion title="Tools Not Available">
    **Problem**: AI tool doesn't show Ref tools
    
    **Solutions**:
    - Verify MCP server configuration
    - Restart AI tool
    - Check server logs for errors
    - Test server connectivity
  </Accordion>

  <Accordion title="Search Returns No Results">
    **Problem**: ref_search_documentation returns empty results
    
    **Solutions**:
    - Try broader search terms
    - Check if documentation is indexed
    - Verify API key and credits
    - Try different query phrasing
  </Accordion>

  <Accordion title="URL Reading Fails">
    **Problem**: ref_read_url can't access URLs
    
    **Solutions**:
    - Verify URL is publicly accessible
    - Check for authentication requirements
    - Try different URL format
    - Ensure content type is supported
  </Accordion>

  <Accordion title="Slow Performance">
    **Problem**: Tools taking too long to respond
    
    **Solutions**:
    - Check network connectivity
    - Verify target sites are responsive
    - Adjust timeout settings
    - Use smaller result limits
  </Accordion>
</AccordionGroup>
