## Prerequisites

Before installing the Ref MCP server:

<AccordionGroup>
  <Accordion title="Node.js Requirement">
    **Required**: Node.js 18.0 or higher
    
    Check your version:
    ```bash
    node --version
    ```
    
    Install or update Node.js:
    - [Download from nodejs.org](https://nodejs.org/)
    - Use a version manager like `nvm`
    - Update via package manager
  </Accordion>

  <Accordion title="Ref API Key">
    **Required**: Valid Ref API key
    
    Get your API key:
    1. Sign up at [ref.tools](https://ref.tools)
    2. Navigate to your dashboard
    3. Copy your API key
    4. Store securely (you'll need it for configuration)
  </Accordion>

  <Accordion title="AI Tool Setup">
    **Required**: Compatible AI tool
    
    Supported tools:
    - Claude Desktop
    - Cursor 
    - VS Code (with MCP extensions)
    - Custom MCP clients
  </Accordion>
</AccordionGroup>

## Method 1: NPM Global Install (Recommended)

The easiest way to install the Ref MCP server:

<Steps>
  <Step title="Install Package">
    ```bash
    npm install -g @ref-tools/mcp-server
    ```
    
    This installs the `ref-mcp-server` command globally.
  </Step>

  <Step title="Verify Installation">
    ```bash
    ref-mcp-server --version
    ```
    
    Should display the current version number.
  </Step>

  <Step title="Test Basic Functionality">
    ```bash
    REF_API_KEY=your-api-key ref-mcp-server --test
    ```
    
    This runs a quick connectivity test.
  </Step>
</Steps>

### Troubleshooting NPM Install

<AccordionGroup>
  <Accordion title="Permission Errors">
    **Problem**: `EACCES` errors during global install
    
    **Solutions**:
    ```bash
    # Option 1: Use npx instead
    npx @ref-tools/mcp-server
    
    # Option 2: Configure npm prefix
    npm config set prefix ~/.npm-global
    export PATH=~/.npm-global/bin:$PATH
    
    # Option 3: Use sudo (not recommended)
    sudo npm install -g @ref-tools/mcp-server
    ```
  </Accordion>

  <Accordion title="Package Not Found">
    **Problem**: Package doesn't exist or name incorrect
    
    **Solutions**:
    ```bash
    # Check if package is published
    npm view @ref-tools/mcp-server
    
    # Try alternative name (verify in documentation)
    npm install -g ref-mcp-server
    
    # Check npm registry
    npm config get registry
    ```
  </Accordion>
</AccordionGroup>

## Method 2: Source Build

Build from source for development or customization:

<Steps>
  <Step title="Clone Repository">
    ```bash
    git clone https://github.com/ref-tools/ref-tools-mcp.git
    cd ref-tools-mcp
    ```
  </Step>

  <Step title="Install Dependencies">
    ```bash
    npm install
    ```
  </Step>

  <Step title="Build Project">
    ```bash
    npm run build
    ```
  </Step>

  <Step title="Link Globally">
    ```bash
    npm link
    ```
    
    This makes `ref-mcp-server` available globally.
  </Step>

  <Step title="Verify Build">
    ```bash
    ref-mcp-server --version
    ```
  </Step>
</Steps>

### Development Mode

For active development:

```bash
# Start in development mode with auto-reload
npm run dev

# Run tests
npm test

# Run linting
npm run lint
```

## Method 3: Docker Container

Run the MCP server in a container:

<Steps>
  <Step title="Pull Image">
    ```bash
    docker pull ref-tools/mcp-server:latest
    ```
  </Step>

  <Step title="Run Container">
    ```bash
    docker run -d \
      --name ref-mcp-server \
      -e REF_API_KEY=your-api-key \
      -p 8080:8080 \
      ref-tools/mcp-server:latest
    ```
  </Step>

  <Step title="Test Connection">
    ```bash
    curl http://localhost:8080/health
    ```
  </Step>
</Steps>

### Docker Compose

For more complex setups:

```yaml
# docker-compose.yml
version: '3.8'
services:
  ref-mcp-server:
    image: ref-tools/mcp-server:latest
    environment:
      - REF_API_KEY=${REF_API_KEY}
      - REF_TIMEOUT=30000
    ports:
      - "8080:8080"
    restart: unless-stopped
    healthcheck:
      test: ["CMD", "curl", "-f", "http://localhost:8080/health"]
      interval: 30s
      timeout: 10s
      retries: 3
```

Start with:
```bash
docker-compose up -d
```

## Method 4: Manual Installation

Download and run directly:

<Steps>
  <Step title="Download Release">
    Download the latest release from:
    ```
    https://github.com/ref-tools/ref-tools-mcp/releases/latest
    ```
  </Step>

  <Step title="Extract Archive">

    ```bash
    cd ref-mcp-server-v1.0.0
    ```
  </Step>

  <Step title="Install Dependencies">
    ```bash
    npm install --production
    ```
  </Step>

  <Step title="Run Server">
    ```bash
    REF_API_KEY=your-api-key node dist/index.js
    ```
  </Step>
</Steps>

## Environment Configuration

Configure the MCP server with environment variables:

```bash
# Required
export REF_API_KEY="your-api-key-here"

# Optional
export REF_API_ENDPOINT="https://ref.tools/api"
export REF_TIMEOUT="30000"
export REF_MAX_RESULTS="10"
export REF_DEBUG="false"
export REF_CACHE_TTL="3600"
```

### Configuration File

Alternatively, use a configuration file:

```json
// ref-config.json
{
  "apiKey": "your-api-key-here",
  "endpoint": "https://ref.tools/api",
  "timeout": 30000,
  "maxResults": 10,
  "debug": false,
  "cacheTTL": 3600
}
```

Run with config file:
```bash
ref-mcp-server --config ref-config.json
```

## Verification

Test your installation:

### Basic Test
```bash
# Set API key
export REF_API_KEY="your-api-key"

# Test server
ref-mcp-server --test

# Expected output:
# ✓ API key valid
# ✓ Connection successful
# ✓ MCP server ready
```

### Interactive Test
```bash
# Start server in test mode
ref-mcp-server --interactive

# Try commands:
# > search "React hooks"
# > read-url "https://react.dev"
# > help
# > exit
```

### Integration Test

Test with a simple MCP client:

```javascript
// test-client.js
import { Client } from '@modelcontextprotocol/sdk/client/index.js';
import { StdioClientTransport } from '@modelcontextprotocol/sdk/client/stdio.js';

const transport = new StdioClientTransport({
  command: 'ref-mcp-server',
  env: { REF_API_KEY: 'your-api-key' }
});

const client = new Client({
  name: "test-client",
  version: "1.0.0"
}, {
  capabilities: {}
});

await client.connect(transport);

const result = await client.callTool({
  name: 'ref_search_documentation',
  arguments: { query: 'React hooks' }
});

console.log(result);
```

Run with:
```bash
node test-client.js
```

## Troubleshooting

<AccordionGroup>
  <Accordion title="Server Won't Start">
    **Symptoms**: Server exits immediately or fails to start
    
    **Debugging**:
    ```bash
    # Enable debug mode
    REF_DEBUG=true ref-mcp-server
    
    # Check logs
    ref-mcp-server --log-level debug
    
    # Verify environment
    ref-mcp-server --check-env
    ```
    
    **Common Causes**:
    - Missing or invalid API key
    - Port already in use
    - Missing dependencies
    - Permission issues
  </Accordion>

  <Accordion title="Connection Issues">
    **Symptoms**: Can't connect to Ref API
    
    **Debugging**:
    ```bash
    # Test API connectivity
    curl -H "Authorization: Bearer $REF_API_KEY" \
         https://ref.tools/api/search \
         -d '{"query":"test"}'
    
    # Check network
    ping ref.tools
    
    # Test with different timeout
    REF_TIMEOUT=60000 ref-mcp-server
    ```
    
    **Common Causes**:
    - Network/firewall restrictions
    - Invalid API key
    - Service outages
    - Timeout too low
  </Accordion>

  <Accordion title="AI Tool Integration">
    **Symptoms**: AI tool can't see MCP server
    
    **Debugging**:
    ```bash
    # Verify server responds to MCP protocol
    echo '{"jsonrpc":"2.0","method":"tools/list","id":1}' | ref-mcp-server
    
    # Check if AI tool can start server
    # (depends on specific tool)
    ```
    
    **Common Causes**:
    - Incorrect configuration path
    - Server not in PATH
    - Permission issues
    - AI tool configuration errors
  </Accordion>
</AccordionGroup>
