## What is MCP?

The Model Context Protocol (MCP) is an open standard that enables AI applications to securely access external data sources and tools. Ref's MCP server provides seamless integration with AI tools like Claude Desktop, VS Code, and Cursor.

## Available Tools

The Ref MCP server provides two main tools:

### ref_search_documentation

Search through indexed documentation with natural language queries.

**Usage**: AI tools automatically use this when you ask documentation questions.

**Examples**:
- "Find React hooks documentation"
- "How do I handle errors in Express.js?"
- "Show me TypeScript interface examples"

### ref_read_url

Extract and read content from any URL.

**Usage**: Ask AI tools to read specific documentation pages or files.

**Examples**:
- "Read the Next.js deployment guide"
- "What does this GitHub README say?"
- "Analyze this blog post about performance"

## How It Works

1. **Install MCP Server**: Add Ref's MCP server to your AI tool
2. **Configure Authentication**: Provide your Ref API key
3. **Start Using**: AI tools automatically access documentation when needed
4. **Seamless Experience**: No manual API calls or complex integration

## Installation

### Quick Install

Install the Ref MCP server globally:

```bash
npm install -g @ref-tools/mcp-server
```

### Configuration Example

Most MCP-compatible tools use this configuration format:

```json
{
  "mcpServers": {
    "ref": {
      "command": "ref-mcp-server",
      "env": {
        "REF_API_KEY": "your-api-key-here"
      }
    }
  }
}
```

## Configuration Options

Customize the MCP server behavior:

```json
{
  "mcpServers": {
    "ref": {
      "command": "ref-mcp-server",
      "env": {
        "REF_API_KEY": "your-api-key",
        "REF_TIMEOUT": "30000",
        "REF_MAX_RESULTS": "10",
        "REF_DEBUG": "false"
      }
    }
  }
}
```

| Environment Variable | Default | Description |
|---------------------|---------|-------------|
| `REF_API_KEY` | Required | Your Ref API key |
| `REF_TIMEOUT` | `30000` | Request timeout in milliseconds |
| `REF_MAX_RESULTS` | `10` | Maximum search results |
| `REF_DEBUG` | `false` | Enable debug logging |

## Team Usage

Teams can share MCP configurations:

```json
{
  "mcpServers": {
    "ref": {
      "command": "ref-mcp-server",
      "env": {
        "REF_API_KEY": "team-api-key",
        "REF_TEAM_ID": "your-team-id"
      }
    }
  }
}
```

Benefits:
- Shared credit pool
- Team-specific documentation access
- Centralized billing and management

## Advanced Usage

### Custom MCP Implementation

Build your own MCP client:

```typescript
import { Client } from '@modelcontextprotocol/sdk/client/index.js';
import { StdioClientTransport } from '@modelcontextprotocol/sdk/client/stdio.js';

const transport = new StdioClientTransport({
  command: 'ref-mcp-server',
  env: { REF_API_KEY: 'your-key' }
});

const client = new Client({ name: "my-client", version: "1.0.0" }, {
  capabilities: {}
});

await client.connect(transport);

// Use Ref tools
const result = await client.callTool({
  name: 'ref_search_documentation',
  arguments: { query: 'React hooks' }
});
```

### Protocol Details

The Ref MCP server implements:
- **Tools**: `ref_search_documentation`, `ref_read_url`
- **Transport**: Stdio and HTTP transports
- **Authentication**: API key via environment variables
- **Error Handling**: Proper MCP error responses
