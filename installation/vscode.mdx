## Overview

VS Code supports Ref integration through Model Context Protocol (MCP) extensions, enabling documentation search directly within your development environment.

<Warning>
MCP support in VS Code requires compatible extensions. The ecosystem is rapidly evolving, so installation steps may vary.
</Warning>

## Prerequisites

- **VS Code** (latest version recommended)
- **Node.js 18+** for running the MCP server
- **MCP Extension** for VS Code (see supported extensions below)
- **Ref API Key** from your [dashboard](https://ref.tools/dashboard)

## Installation Methods

### Method 1: Using MCP Configuration File

<Steps>
  <Step title="Install MCP Server">
    ```bash
    npm install -g @ref-tools/mcp-server
    ```
  </Step>

  <Step title="Create MCP Configuration">
    Create a `.mcp.json` file in your workspace root:
    
    ```json
    {
      "mcpServers": {
        "ref": {
          "command": "ref-mcp-server",
          "env": {
            "REF_API_KEY": "your-api-key-here"
          }
        }
      }
    }
    ```
  </Step>

  <Step title="Install MCP Extension">
    Search for and install an MCP-compatible extension from the VS Code marketplace
  </Step>

  <Step title="Configure Extension">
    Configure the extension to use your `.mcp.json` file
  </Step>
</Steps>

### Method 2: Extension-Specific Setup

Some extensions have their own configuration methods:

<AccordionGroup>
  <Accordion title="Claude Dev Extension">
    1. Install the Claude Dev extension
    2. Open extension settings
    3. Add MCP server configuration:
    ```json
    {
      "name": "ref",
      "serverPath": "ref-mcp-server", 
      "env": {
        "REF_API_KEY": "your-api-key"
      }
    }
    ```
  </Accordion>

  <Accordion title="Custom MCP Client">
    For custom MCP implementations:
    1. Follow the specific extension's documentation
    2. Use the Ref MCP server endpoint
    3. Configure authentication with your API key
  </Accordion>
</AccordionGroup>

### Method 3: Direct Integration

For advanced users, integrate directly with VS Code's extension API:

```javascript
// Extension development example
const { spawn } = require('child_process');

// Start Ref MCP server
const refServer = spawn('ref-mcp-server', [], {
  env: { 
    ...process.env,
    REF_API_KEY: 'your-api-key-here'
  }
});

// Handle MCP communication
refServer.stdout.on('data', (data) => {
  // Process MCP messages
});
```

## Configuration Options

### Environment Variables

Configure Ref's behavior:

```json
{
  "mcpServers": {
    "ref": {
      "command": "ref-mcp-server",
      "env": {
        "REF_API_KEY": "your-api-key-here",
        "REF_TIMEOUT": "30000",
        "REF_MAX_RESULTS": "10",
        "REF_CACHE_TTL": "3600"
      }
    }
  }
}
```

### VS Code Settings

Add Ref-specific settings to your VS Code configuration:

```json
{
  "ref.enabled": true,
  "ref.autoSearch": true,
  "ref.searchShortcut": "Ctrl+Shift+R",
  "ref.maxResults": 10
}
```

## Extension Development

### Creating Custom Integration

Build your own VS Code extension with Ref:

<Steps>
  <Step title="Initialize Extension">
    ```bash
    yo code
    # Choose "New Extension (TypeScript)"
    ```
  </Step>

  <Step title="Add Ref Dependency">
    ```bash
    npm install @ref-tools/mcp-client
    ```
  </Step>

  <Step title="Implement Ref Client">
    ```typescript
    import { RefClient } from '@ref-tools/mcp-client';
    
    const client = new RefClient({
      apiKey: process.env.REF_API_KEY
    });
    
    // Register command
    vscode.commands.registerCommand('myext.searchRef', async () => {
      const query = await vscode.window.showInputBox({
        prompt: 'Search documentation'
      });
      
      const results = await client.search(query);
      // Display results
    });
    ```
  </Step>

  <Step title="Package Extension">
    ```bash
    vsce package
    ```
  </Step>
</Steps>

### Extension Manifest

Configure your extension to use Ref:

```json
// package.json
{
  "contributes": {
    "commands": [
      {
        "command": "myext.searchRef",
        "title": "Search Documentation with Ref"
      }
    ],
    "keybindings": [
      {
        "command": "myext.searchRef", 
        "key": "ctrl+shift+r"
      }
    ],
    "configuration": {
      "properties": {
        "myext.refApiKey": {
          "type": "string",
          "description": "Ref API Key"
        }
      }
    }
  }
}
```

## Alternative Integration Methods

### REST API Integration

If MCP isn't suitable, use direct REST API calls:

```typescript
// VS Code extension using REST API
import * as vscode from 'vscode';
import axios from 'axios';

async function searchRef(query: string) {
  const response = await axios.post('https://ref.tools/api/search', {
    query: query
  }, {
    headers: {
      'Authorization': `Bearer ${apiKey}`,
      'Content-Type': 'application/json'
    }
  });
  
  return response.data;
}
```

### WebView Integration

Create a custom webview panel for Ref:

```typescript
// Create webview panel
const panel = vscode.window.createWebviewPanel(
  'refSearch',
  'Ref Documentation Search',
  vscode.ViewColumn.Beside,
  {
    enableScripts: true
  }
);

// Load Ref web interface
panel.webview.html = getWebviewContent();
```

## Troubleshooting

<AccordionGroup>
  <Accordion title="MCP Server Not Starting">
    **Symptoms:** Extension can't connect to Ref
    
    **Solutions:**
    - Verify Node.js 18+ is installed
    - Check MCP server is globally installed
    - Review VS Code output panel for errors
    - Try manual server startup: `ref-mcp-server`
  </Accordion>

  <Accordion title="Extension Not Found">
    **Symptoms:** Can't find MCP-compatible extensions
    
    **Solutions:**
    - Search for "MCP" or "Model Context Protocol" in marketplace
    - Check extension documentation for MCP support
    - Consider using REST API integration instead
    - Join VS Code community for latest extension recommendations
  </Accordion>

  <Accordion title="Commands Not Working">
    **Symptoms:** Ref commands don't appear or don't work
    
    **Solutions:**
    - Reload VS Code window
    - Check extension is enabled and active
    - Verify API key is configured correctly
    - Review developer console for JavaScript errors
  </Accordion>

  <Accordion title="Authentication Issues">
    **Symptoms:** "Unauthorized" or API key errors
    
    **Solutions:**
    - Verify API key is correct and complete
    - Check environment variables are set properly
    - Try generating a new API key
    - Ensure no extra quotes or spaces in configuration
  </Accordion>
</AccordionGroup>
