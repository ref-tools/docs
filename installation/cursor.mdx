## Quick Installation (Recommended)

### Method 1: Deeplink Installation

The fastest way to get Ref working in Cursor:

<Steps>
  <Step title="Get Your API Key">
    Copy your API key from the [Ref dashboard](https://ref.tools/dashboard)
  </Step>

  <Step title="Click to Install">
    Click this button to install Ref in Cursor:
    
    <Card title="Install Ref in Cursor" href="cursor://install-mcp?name=ref&package=@ref-tools/mcp-server&env=REF_API_KEY">
      üöÄ One-Click Install in Cursor
    </Card>
    
    *This will open Cursor and automatically configure Ref for you.*
  </Step>

  <Step title="Enter API Key">
    When prompted, paste your API key and confirm the installation
  </Step>

  <Step title="Start Using">
    Ref is now available in Cursor! Try asking about documentation in your conversations.
  </Step>
</Steps>

<Tip>
Make sure Cursor is installed and running before clicking the install button.
</Tip>

## Manual Installation

If deeplinks don't work, you can install manually:

<Steps>
  <Step title="Open Cursor Settings">
    Navigate to Cursor Settings ‚Üí Extensions ‚Üí MCP Servers
  </Step>

  <Step title="Add New MCP Server">
    Click "Add Server" and enter:
    - **Name:** `ref`
    - **Package:** `@ref-tools/mcp-server`
    - **Command:** `ref-mcp-server`
  </Step>

  <Step title="Configure Environment">
    Add environment variable:
    - **Key:** `REF_API_KEY`
    - **Value:** `your-api-key-here`
  </Step>

  <Step title="Save and Restart">
    Save the configuration and restart Cursor
  </Step>
</Steps>

## Cursor Configuration File

Alternatively, edit Cursor's MCP configuration directly:

<Steps>
  <Step title="Open Config File">
    Open Cursor's MCP configuration file:
    
    **Location varies by OS, typically:**
    - macOS: `~/Library/Application Support/Cursor/mcp_config.json`
    - Windows: `%APPDATA%\Cursor\mcp_config.json`
    - Linux: `~/.config/Cursor/mcp_config.json`
  </Step>

  <Step title="Add Ref Configuration">
    ```json
    {
      "mcpServers": {
        "ref": {
          "command": "npx",
          "args": ["@ref-tools/mcp-server"],
          "env": {
            "REF_API_KEY": "your-api-key-here"
          }
        }
      }
    }
    ```
  </Step>

  <Step title="Restart Cursor">
    Close and restart Cursor to load the new configuration
  </Step>
</Steps>

## Advanced Configuration

### Team Setup

For teams using Ref, configure team-specific access:

```json
{
  "mcpServers": {
    "ref": {
      "command": "npx",
      "args": ["@ref-tools/mcp-server"],
      "env": {
        "REF_API_KEY": "team-api-key-here",
        "REF_TEAM_ID": "your-team-id"
      }
    }
  }
}
```

### Custom Configuration

Customize Ref's behavior in Cursor:

```json
{
  "mcpServers": {
    "ref": {
      "command": "npx",
      "args": ["@ref-tools/mcp-server"],
      "env": {
        "REF_API_KEY": "your-api-key",
        "REF_MAX_RESULTS": "15",
        "REF_TIMEOUT": "45000",
        "REF_CACHE_ENABLED": "true"
      }
    }
  }
}
```

## Troubleshooting

<AccordionGroup>
  <Accordion title="Deeplink Not Working">
    **Symptoms:** Clicking install button does nothing
    
    **Solutions:**
    - Ensure Cursor is installed and running
    - Try copying the deeplink URL and opening it manually
    - Check if your browser blocks custom URL schemes
    - Use manual installation method instead
  </Accordion>

  <Accordion title="MCP Server Not Found">
    **Symptoms:** "ref-mcp-server command not found" error
    
    **Solutions:**
    - Install globally: `npm install -g @ref-tools/mcp-server`
    - Use npx in command: `npx @ref-tools/mcp-server`
    - Check Node.js is installed: `node --version`
    - Verify npm is in PATH
  </Accordion>

  <Accordion title="API Key Issues">
    **Symptoms:** Authentication errors in Cursor
    
    **Solutions:**
    - Double-check API key is copied correctly
    - Verify no extra spaces or characters
    - Generate a new API key if needed
    - Check API key hasn't expired
  </Accordion>

  <Accordion title="No Search Results">
    **Symptoms:** Cursor can't find documentation
    
    **Solutions:**
    - Check credit balance in dashboard
    - Try more specific search terms
    - Verify internet connection
    - Check if documentation is indexed in Ref
  </Accordion>
</AccordionGroup>

## Updates and Maintenance

### Automatic Updates

Cursor will typically update MCP servers automatically, but you can force updates:

```bash
npm update -g @ref-tools/mcp-server
```

### Version Management

Check your current version:

```bash
npm list -g @ref-tools/mcp-server
```

### Configuration Backup

Save your configuration for team sharing:

```bash
# Export configuration
cp ~/.config/Cursor/mcp_config.json ref-cursor-config.json

# Share with team or backup
git add ref-cursor-config.json
```
