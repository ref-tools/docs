## Why Ref

- Ref keeps agents on the latest docs.
- It trims context to the relevant 5k tokens.
- Sessions track prior searches to avoid repeats.
- Works with MCP compliant editors and CLIs.

## Core Tools

- `ref_search_documentation` finds precise docs fast.
- `ref_read_url` converts any URL into clean markdown.
- OpenAI deep research maps to `search` and `fetch` wrappers.

## Supported Content Sources

- Public web docs across services.
- Private GitHub repos and large mono repos.
- PDFs and Markdown knowledge bases when indexing is enabled.
