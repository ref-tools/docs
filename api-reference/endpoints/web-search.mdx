## Endpoint

```
POST https://ref.tools/api/web-search
```

Search the web for real-time information beyond indexed documentation. Perfect for finding recent updates, current events, and fresh content not yet in documentation indices.

## Request

### Headers

| Header | Value |
|--------|-------|
| `Authorization` | `Bearer YOUR_API_KEY` |
| `Content-Type` | `application/json` |

### Body Parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `query` | string | Yes | Search query |
| `maxResults` | integer | No | Maximum results to return (default: 5, max: 20) |
| `freshness` | string | No | Freshness filter: `day`, `week`, `month`, `year` |
| `language` | string | No | Language code (e.g., `en`, `es`, `fr`) (default: `en`) |
| `region` | string | No | Region code for localized results (e.g., `us`, `uk`, `ca`) |

### Example Request

```bash
curl -X POST https://ref.tools/api/web-search \
  -H "Authorization: Bearer ref_api_1234567890abcdef" \
  -H "Content-Type: application/json" \
  -d '{
    "query": "Next.js 15 new features 2024",
    "maxResults": 5,
    "freshness": "month",
    "language": "en"
  }'
```

## vs Documentation Search

| Feature | Web Search | Documentation Search |
|---------|------------|---------------------|
| **Content** | Real-time web content | Indexed documentation |
| **Freshness** | Latest information | May lag behind updates |
| **Quality** | Variable quality | Curated, high-quality |
| **Speed** | Slower (external APIs) | Faster (internal index) |
| **Cost** | 2 credits | 1 credit |
| **Coverage** | Entire web | Specific docs only |

### When to Use Each

**Use Web Search for**:
- Breaking news and updates
- Recent releases and announcements  
- Current issues and solutions
- Comparative analysis with latest data
- Content not yet indexed

**Use Documentation Search for**:
- Established APIs and frameworks
- Detailed technical documentation
- Code examples and tutorials
- Comprehensive guides
- Fast, reliable information

## Limitations

### Rate Limits
- Higher cost: **2 credits per request**
- Same rate limits as other endpoints
- Consider impact on credit budget

### Content Quality
- Results may include low-quality content
- Not all sources are technically accurate
- May include outdated information despite freshness filters
- Requires human verification for critical decisions

### Coverage
- Limited to publicly accessible content
- May miss some specialized or subscription-based sources
- Language and region availability varies
- Some content may be geo-blocked

## Integration Examples

### Hybrid Search Strategy

```javascript
async function comprehensiveSearch(query) {
  // Try documentation search first (cheaper, faster)
  const docResults = await client.search({
    query: query,
    maxResults: 5
  });
  
  // If insufficient results, supplement with web search
  if (docResults.results.length < 3) {
    const webResults = await client.webSearch({
      query: query + " 2024",
      maxResults: 5,
      freshness: "month"
    });
    
    return {
      documentation: docResults.results,
      web: webResults.results
    };
  }
  
  return { documentation: docResults.results };
}
```

### News Monitoring

```javascript
async function monitorTechNews(technologies) {
  const results = {};
  
  for (const tech of technologies) {
    const news = await client.webSearch({
      query: `${tech} updates news 2024`,
      freshness: "week",
      maxResults: 3
    });
    
    results[tech] = news.results;
  }
  
  return results;
}
```
