## Base URL

All API requests should be made to:

```
https://ref.tools/api
```

## Authentication

All API endpoints require authentication using Bearer tokens in the Authorization header:

```bash
curl -X POST https://ref.tools/api/search \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{"query": "React hooks"}'
```

Get your API key from the [Ref dashboard](https://ref.tools/dashboard).

## Request Format

All requests must include:
- `Content-Type: application/json` header
- `Authorization: Bearer {api_key}` header
- JSON request body for POST requests

## Response Format

All responses return JSON with this structure:

```json
{
  "results": [...],          // Main response data
  "totalResults": 42,        // Total number of results (if applicable)
  "searchTime": 234,         // Response time in milliseconds
  "creditsUsed": 1,          // Credits consumed by this request
  "requestId": "req_12345"   // Unique request identifier
}
```

## Rate Limits

API requests are rate-limited based on your plan:

| Plan | Requests/Hour | Burst Limit |
|------|---------------|-------------|
| Free | 100 | 10/minute |
| Basic | 1,000 | 50/minute |
| Pro | 5,000 | 100/minute |
| Team | 10,000 | 200/minute |

Rate limit information is included in response headers:

```
X-RateLimit-Limit: 1000
X-RateLimit-Remaining: 999
X-RateLimit-Reset: 1640995200
```

## Error Handling

The API uses standard HTTP status codes:

| Status Code | Meaning |
|-------------|---------|
| 200 | Success |
| 400 | Bad Request - Invalid parameters |
| 401 | Unauthorized - Invalid API key |
| 402 | Payment Required - Insufficient credits |
| 429 | Too Many Requests - Rate limit exceeded |
| 500 | Internal Server Error |

Error responses include detailed information:

```json
{
  "error": "insufficient_credits",
  "message": "Not enough credits for this operation",
  "code": "INSUFFICIENT_CREDITS",
  "creditsRequired": 2,
  "creditsAvailable": 0
}
```

## Getting Started

1. **Get an API Key**: Sign up at [ref.tools](https://ref.tools) and copy your API key
2. **Make a Test Request**: Try the search endpoint with a simple query
3. **Integrate**: Use our SDKs or make direct HTTP requests
4. **Monitor Usage**: Track your credit usage in the dashboard

Ready to start? Check out the [search endpoint](/api-reference/endpoints/search) documentation.
