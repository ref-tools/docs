## Overview

Ref's indexing system allows you to add custom documentation sources to your searchable index. Whether it's your company's internal docs, a specific GitHub repository, or PDF documentation, Ref can crawl, process, and make it searchable.

## Supported Sources

### Web Documentation Sites

Index entire documentation websites or specific sections:

**Supported formats:**
- Static site generators (Docusaurus, GitBook, VitePress, etc.)
- Wiki systems (Confluence, Notion, MediaWiki)
- Custom HTML documentation
- API documentation sites

**Example sites:**
- https://docs.react.dev
- https://nextjs.org/docs
- https://docs.python.org
- Your internal company documentation

### GitHub Repositories

Index code repositories to make documentation searchable:

**What gets indexed:**
- README.md files
- Documentation folders (/docs, /documentation)
- Wiki pages
- Markdown files throughout the repository
- Code comments and docstrings

**Repository types:**
- Public repositories (free indexing)
- Private repositories (requires authentication)
- Organization repositories
- Forked repositories

### PDF Documents

Process PDF documentation for searchability:

**Supported features:**
- Text extraction from PDF files
- OCR for image-based PDFs
- Multi-page document processing
- Metadata extraction

**Best practices:**
- Text-based PDFs work better than image PDFs
- Clear document structure improves search results
- Smaller files (< 50MB) process faster

## Indexing Process

### Starting an Index Job

<CodeGroup>

```bash Web Documentation
curl -X POST https://ref.tools/api/index \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "type": "web",
    "url": "https://docs.yourproject.com",
    "options": {
      "maxPages": 1000,
      "followSubdomains": false
    }
  }'
```

```bash GitHub Repository
curl -X POST https://ref.tools/api/index \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "type": "github",
    "url": "https://github.com/owner/repo",
    "options": {
      "includeBranches": ["main", "develop"],
      "includeWiki": true
    }
  }'
```

```bash PDF Document
curl -X POST https://ref.tools/api/index \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "type": "pdf",
    "url": "https://storage.googleapis.com/your-bucket/manual.pdf"
  }'
```

</CodeGroup>

### Index Job Response

## Index Management

### Monitoring Index Jobs

Track the progress of your indexing jobs:

<CodeGroup>

```bash Check Job Status
curl -X GET https://ref.tools/api/index/job_abc123 \
  -H "Authorization: Bearer YOUR_API_KEY"
```

```json Response
{
  "jobId": "job_abc123",
  "status": "completed",
  "progress": {
    "pagesProcessed": 247,
    "totalPages": 247,
    "duration": "8 minutes"
  },
  "results": {
    "documentsIndexed": 247,
    "bytesProcessed": 2485760,
    "averagePageSize": 10065
  }
}
```

</CodeGroup>

### Updating Existing Indices

Keep your documentation up-to-date with re-indexing:

```bash
curl -X POST https://ref.tools/api/index/update \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "indexId": "index_xyz789",
    "forceRefresh": true
  }'
```

<Tip>
Ref automatically detects changes in web documentation and GitHub repositories. Manual updates are only needed for major structural changes.
</Tip>

### Deleting Indices

Remove documentation that's no longer needed:

```bash
curl -X DELETE https://ref.tools/api/index/index_xyz789 \
  -H "Authorization: Bearer YOUR_API_KEY"
```

<Warning>
Deleting an index is permanent and cannot be undone. Make sure you no longer need the indexed documentation.
</Warning>

## Advanced Indexing Options

### Web Crawling Configuration

Fine-tune how web documentation is crawled:

```json
{
  "type": "web",
  "url": "https://docs.yourproject.com",
  "options": {
    "maxPages": 1000,
    "maxDepth": 5,
    "followSubdomains": false,
    "includePatterns": ["/docs/*", "/api/*"],
    "excludePatterns": ["/blog/*", "/news/*"],
    "respectRobotsTxt": true,
    "waitTime": 1000,
    "userAgent": "Ref Documentation Indexer"
  }
}
```

### GitHub Integration Options

Customize GitHub repository indexing:

```json
{
  "type": "github",
  "url": "https://github.com/owner/repo",
  "options": {
    "includeBranches": ["main", "develop", "docs"],
    "includeWiki": true,
    "includeIssues": false,
    "includePullRequests": false,
    "filePatterns": ["*.md", "*.rst", "*.txt"],
    "excludeDirectories": ["node_modules", ".git", "build"]
  }
}
```

### Content Quality

Improve search results by optimizing your source documentation:

<CardGroup cols={2}>

  </Card>
  
  <Card title="Rich Metadata">

### Performance Optimization

Optimize indexing performance:

- **Smaller Pages**: Break large documents into smaller, focused pages
- **Clean HTML**: Use semantic HTML and avoid excessive styling
- **Fast Loading**: Ensure your documentation site loads quickly
- **Sitemap**: Provide XML sitemaps for better crawl efficiency

## Team Indexing

### Team-Specific Indices

Teams can create private indices for internal documentation:

```bash
curl -X POST https://ref.tools/api/teams/{team_id}/index \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "type": "web",
    "url": "https://internal-docs.company.com",
    "name": "Company Internal Documentation",
    "visibility": "team"
  }'
```

### Access Control

Control who can access team indices:

- **Team-Only**: Only team members can search this documentation
- **Organization**: All organization members have access
- **Public**: Anyone with the link can access (useful for public projects)

## Credit Usage

Understanding indexing costs:

| Source Type | Credit Cost | Notes |
|-------------|-------------|-------|
| **Web Documentation** | 10 credits | One-time cost per site |
| **GitHub Repository** | 0 credits | Free for public repositories |
| **PDF Document** | 0 credits | Free processing |
| **Private GitHub** | 0 credits | Free with valid authentication |
| **Re-indexing** | Same as initial | Full cost for complete re-index |
| **Update Index** | Variable | Only changed content is processed |

<Info>
Indexing is a one-time cost. Once your documentation is indexed, searching it only uses standard search credits (1 credit per search).
</Info>

## Best Practices

### Before Indexing

<Steps>
  <Step title="Review Content">
    Ensure your documentation is complete and up-to-date
  </Step>
  
  <Step title="Check Access">
    Verify that Ref can access all pages you want to index
  </Step>
  
  <Step title="Plan Structure">
    Organize content in a logical hierarchy for better search results
  </Step>
  
  <Step title="Test Connectivity">
    Make sure URLs are accessible and not behind authentication
  </Step>
</Steps>

### After Indexing

- **Test Search**: Try various search queries to verify content is discoverable
- **Monitor Usage**: Track which documentation gets searched most
- **Update Regularly**: Re-index when significant changes are made
- **Gather Feedback**: Ask team members about search result quality

## Troubleshooting

<AccordionGroup>
  <Accordion title="Indexing Failed">
    - Check if the URL is accessible
    - Verify authentication credentials
    - Ensure the site allows crawling (robots.txt)
    - Contact support for persistent failures
  </Accordion>

  <Accordion title="Missing Pages">
    - Check if pages are linked from the main navigation
    - Verify pages aren't excluded by robots.txt
    - Ensure pages load without JavaScript requirements
    - Review include/exclude patterns in your configuration
  </Accordion>

  <Accordion title="Poor Search Results">
    - Re-index if content has changed significantly
    - Improve page titles and headings in source documentation
    - Add more descriptive content and keywords
    - Break large pages into smaller, focused sections
  </Accordion>

  <Accordion title="Authentication Issues">
    - Verify credentials are correct and have sufficient permissions
    - Check if tokens have expired
    - Ensure API keys have read access to the repository/site
    - Test authentication separately before indexing
  </Accordion>
</AccordionGroup>
